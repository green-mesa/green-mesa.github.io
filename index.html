    Hyperbone

    Hyperbone is a Hypermedia Framework, built on top of Backbone.js, for building client side applications.

    It natively consumes JSON HAL and adds custom extensions, _commands which can come from the server (or be added to your model prototypes in the client) to enable definition of server side interactions, which can then be bound to particular forms.

    Unlike Backbone, Hyperbone's evented models are fully nested - It's Hyperbone all the way down - and support dot notation accessors.

    Also unlike Backbone, the view engine is far more like Angular's, in that it binds your Hypermedia models directly to the DOM. It supports templated attributes and innerText and a very very small number of custom attributes.

    It tries to stay true to Backbone's original mission which is to keep your truth out of the DOM. Properly built Hyperbone models mean that your application only manipulates the model, never the DOM. Any actual DOM level tinkering that's required should be done via View extensions, for which there's a neato API.

    The primary side effect of this framework seems to be that you can build a UI for a server side that doesn't yet exist, using _commands in the model prototype to simulate the desired API which server-side developers can then use as a well defined spec to build an API to. This seems pretty interesting to me, as I've not been able to work this way before. This will form a core part of the documentation for Hyperbone!

    There's no one single Hyperbone component yet. Each part of the Framework is distributed via the Component package manager. Use one or all of the components as you see fit.

    Activity on this project is high at the moment.

    Hyperbone Model

    Evented Hypermedia Models with an API pretty familiar to Backbone users.

    Hyperbone View

    Bind Hyperbone Models to the DOM, with special Hypermedia conventions.

    Hyperbone View Commands

    Bind forms in your DOM to specific Hypermedia commands. Bind to Hyperbone Submit events. Where things start getting "interesting".

    Coming Soon: Hyperbone IO

    HyperboneIO deals with network interactions for Hypermedia documents - loading, reloading and sending _command data to the server. By keeping as a separate module, you're at liberty to use whatever network interaction stuff you like.

    Coming Soon: Hyperbone Router

    Routing. Note how by doing this as a standalone module you can replace this with something else.

    Coming Soon: Hyperbone

    One single component to add to your project that exposes the full Hyperbone experience.

    Backbone

    We have broken Backbone into discrete standalone modules so that Hyperbone can avoid loading unnecessary parts.
